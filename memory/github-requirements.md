## 📋 发布流程要求

### 1. 开发阶段

- ✅ **加新功能先不推送** - 每次添加新功能后先本地提交，不推送到 GitHub
- ✅ **累积多个功能后再推送** - 等用户明确说"推送"时再一起推送
- ✅ **版本号管理** - 在用户说推送前，都用同一个版本号（如 v1.6.0）
- ✅ **推送后更新版本号** - 用户确认推送后，再创建新的 Git Tag 和 Release

### 2. 隐私保护

- ❌ **不要泄露真实 MAC 地址** - Release 说明、README、文档中使用示例 MAC
- ❌ **不要泄露真实 IP 地址** - 使用示例 IP（如 192.168.1.x、10.0.0.x）
- ❌ **不要泄露真实密码** - 配置示例中使用"空密码"或"yourpassword"

### 3. 示例格式

**正确的 MAC 地址示例**:
```
00:0C:42:AA:BB:CC  ← MikroTik OUI
4C:5E:0C:DD:EE:FF  ← MikroTik OUI
D4:CA:6D:11:22:33  ← MikroTik OUI
```

**正确的 IP 地址示例**:
```
192.168.1.1      ← 网关
192.168.1.10     ← 设备 1
192.168.1.20     ← 设备 2
10.0.0.1         ← 内网网关
```
